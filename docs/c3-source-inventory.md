## Honest caveat (for paper)

These are **teacher-generated** from Qwen3-Coder-480B during the KIKI stack-training pipeline, NOT user-collected real dialogues. The honest framing for Paper A §6 "Limitations":

> The "real corpus" used in C3 is teacher-generated by Qwen3-Coder-480B, representative of the query distribution the model would encounter in its intended deployment (technical expert Q&A), but not sampled from actual end-user interactions. A fully user-collected corpus remains future work.

This is still a massive upgrade over the `data/final/` synthetic corpus (pure prompt-template generation), because mascarade-datasets:
- Have multi-turn structure (system/human/gpt)
- Include realistic follow-up questions
- Cover failure modes + edge cases the teacher raised

## Ingestion note for `build_corpus_real.py`

The `_ingest_source` function extracts user-authored turns. For mascarade schema:

```python
# for conv in jsonl: for turn in conv["conversations"]: if turn["from"] == "human": yield turn["value"]
```

Implementation note in `scripts/build_corpus_real.py` must handle this schema. The C3 plan's current `_ingest_source` handles `{role, content}` with `role == "user"` and `{question, ...}` — it needs a third branch for `{conversations: [{from, value}]}`.
